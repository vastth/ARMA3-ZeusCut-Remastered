# 剧情密度规范

> **定位**：指导 AI 续写和导演规划时，控制单集（2小时直播）内的内容密度。
> 基于 S1-1 至 S1-4 的实际执行数据归纳，持续迭代。

---

## 一、2小时单集的合理承载上限

| 维度 | 上限 | 说明 |
|------|------|------|
| 主要场景数 | 4-5个 | 超过5个场景会导致每个场景呼吸时间不足 |
| 战斗场次 | 1-2场 | 每场战斗含Zeus操作时间，不低于20分钟 |
| 新引入NPC（完整关系线） | 1个 | 超过1个新NPC会分散关系建立的戏剧重量 |
| 新建伏笔 | 1-2条 | 新伏笔必须在本集内有第一次实质落点，不能只是「提及」 |
| 并行主线数 | 1条主 + 1条背景 | 两条平权主线会产生双线切换成本 |

---

## 二、单集结构模板（已验证最稳定版）

```
第零幕（可选）：承接上集的直接后果，快速收尾上集遗留悬念（控制在10分钟内）

第一幕：本集主线的触发事件（一个具体的行动起点）
  └─ 节点1 → 节点2 → 节点3（因果链，每个节点是上一个的直接后果）

第二幕：主线推进中途出现新信息/新障碍（不是新支线，是主线的转折）
  └─ 节点4 → 节点5

结尾：主线有明确结果（完成或失败均可），加一个情感落点或悬念钩子
```

**关键约束**：每个节点必须是上一个节点的**直接后果**。不允许出现「过了一段时间」「导演根据节奏控制」的时间跳跃。

---

## 三、密度分级与对应策略

### A级集（世界观建立集 / 季首集）
- **特征**：需要建立世界观、势力格局、核心人物
- **密度上限**：主动压低战斗数量（最多1场），把时间留给对话和信息传递
- **AI续写注意**：不要在第一集塞超过2条主要关系线；展示实力应在有赌注的场景内完成，不单独设计热身节点

### B级集（渗透/任务推进集 / 中期集）
- **特征**：主角在某势力内站稳脚跟，完成具体任务
- **密度上限**：1条主线驱动（任务线），1条背景线（关系变化）
- **AI续写注意**：新NPC关系线开启后必须在3集内有实质使用，否则不建立

### C级集（内部清洗/晋升集）
- **特征**：处理内部矛盾，地位发生变化
- **密度上限**：聚焦单一因果链（失败→归责→清除→晋升），外部任务作为触发器而非主戏
- **AI续写注意**：这类集最适合用「情感落点型」结尾，而不是悬念钩子

### D级集（季中/季末集）
- **特征**：多条伏笔集中兑现，关系格局重组
- **密度上限**：可以例外地推进2条主线，但必须在设计时明确两条线的交汇点
- **AI续写注意**：两条线必须在本集内产生交集，不允许「平行推进、各自收尾」

## 四、结尾类型轮换规则

> 连续使用同一种结尾类型超过3集，观众会开始预期，冲击力递减。

| 结尾类型 | 效果 | 使用频率建议 |
|---------|------|------------|
| 悬念钩子型 | 驱动下集期待，制造未完成感 | 每季最多5次 |
| 情感落点型 | 完成某件事/关系转变，给观众喘息 | 每季至少3次 |
| 反转揭露型 | 已知信息被颠覆，世界观重组 | 每季1-2次（季中/季末保留） |

**当前状态（截至S1-4）**：已连续使用悬念钩子型4次。**S1-5建议使用情感落点型结尾。**

## 五、NPC管理规则

### 引入成本规则
- 新NPC完整登场（有名字、有台词、有关系线）= 占用约15-20分钟直播时间
- 每集最多完整引入1个新NPC；其余新面孔只能以「匿名功能性角色」出现

### 3集使用规则
- 任何在剧本中建立了关系线的NPC，必须在**3集内**有一次实质性使用（不是「提及」）
- 超过3集未使用的NPC关系线，视为休眠，写入 `L2_当前状态层/人物当前状态.md` 的休眠区

### 当前需监控的NPC
- **阿虎**：S1-2建立欠人情，截至S1-4未使用 → S1-5或S1-6必须使用，否则从活跃关系中移除

## 六、AI续写强制检查项

在生成任何场次剧本前，必须确认以下项：

- [ ] 本集场景数是否超过5个？（超过则合并）
- [ ] 本集战斗数是否超过2场？（超过则删减）
- [ ] 本集是否有超过1个完整新NPC引入？（超过则推迟）
- [ ] 本集是否有2条平权主线同时推进？（有则降一条为背景线）
- [ ] 结尾类型与上一集是否相同？（相同则考虑轮换）
- [ ] 所有节点是否都是上一个节点的直接后果？（有时间跳跃则修改）
- [ ] 新建伏笔是否在本集有实质落点？（没有则不新建）

# Kanban + Dataview 组合构建技能

> ⚠️ 本技能内含从实际错误中总结的规范，必须严格遵守。
> 详细错误记录见 `.claude/kanban-lessons.md`

---

## 核心规范（禁止违反）

### Kanban 文件铁律

| 规则 | ✅ 正确 | ❌ 绝对禁止 |
|------|---------|------------|
| Settings JSON 格式 | 裸写在 `%% kanban:settings` 和 `%%` 之间 | 用反引号 ` ``` ` 包裹 JSON |
| Frontmatter 字段 | **只保留 `kanban-plugin: board`** | 加入中文字段（场次、标题等）|
| Dataview 块 | 放在**独立配套文件**中 | 写在 Kanban 文件内 |
| Settings 复杂配置 | 让 Obsidian UI 自动生成 | 手写 metadata-keys 数组 |
| 空列 | `## 列名`（下方空行）| 加 HTML 注释占位 |

---

## 正确的 Kanban 文件模板

```markdown
---
kanban-plugin: board
---

## 列名一

- [ ] 卡片文本 #tag

## 列名二

- [ ] 另一张卡片

%% kanban:settings
{"kanban-plugin":"board","lane-width":300}
%%
```

**注意**：
- `lane-width` 是可选的，值为像素数（建议 260-320）
- 卡片支持 `#tag` 标签、`[[wiki链接]]`、`**加粗**` 文本
- 卡片完成用 `- [x]`

## 触发场景

| 用户说 | 执行模式 |
|--------|---------|
| `"新建主进度看板"` | → **模式A：全局场次进度看板** |
| `"新建场次子看板 S1-3"` | → **模式B：单场工作流看板** |
| `"新建追踪表 S1-3"` | → **模式C：Dataview 追踪表** |
| `"建看板套件 S1-3"` | → **模式D：看板 + 追踪表 完整套件** |
| `"更新主看板，添加 S1-3"` | → **模式E：更新现有看板** |

## 模式A：全局场次进度看板

**位置**：`L5_场次工作区/📋场次进度看板.md`
**用途**：全季场次三列进度总览

### 步骤
1. 读取 `L5_场次工作区/` 下所有场次文件夹，列出现有场次
2. 读取各场次 `单场摘要.md` 的 `执行状态` 字段，分配到对应列
3. 写入文件（严格按模板格式，不加任何额外 frontmatter 字段）

### 输出格式
```markdown
---
kanban-plugin: board
---

## 🔴 规划中

- [ ] **S{X}-{N}**《{标题}》· {一句话描述} #{场次tag} #规划中

## ✅ 已完成

%% kanban:settings
{"kanban-plugin":"board","lane-width":380}
%%
```

> ⚠️ 主进度看板只有两列：**规划中** 和 **已完成**。
> 「直播中」列不需要——直播时导演只看单场摘要/剧本/速查手册，不看进度看板。

## 模式B：单场工作流看板

**位置**：`L5_场次工作区/S{X}/S{X}-{N}/场次子看板.md`
**用途**：单场次直播前→中→后→归档工作流

### 步骤
1. 确认目标场次（如 S1-3）
2. 按以下格式创建文件（仅替换 tag 中的场次编号）

### 输出格式
```markdown
---
kanban-plugin: board
---

## 📋 待准备（直播前）

- [ ] 确认剧本定稿 #{场次tag} #待准备
- [ ] 检查活跃伏笔 #{场次tag} #待准备
- [ ] 冻结开场快照 #{场次tag} #待准备

## 📝 直播后（1小时内）

- [ ] 填写状态变更单 #{场次tag} #直播后
- [ ] 更新伏笔推进记录 #{场次tag} #直播后
- [ ] 同步 L2 状态层 #{场次tag} #直播后

## ✅ 归档完成

- [ ] 完成导演复盘卡 #{场次tag} #归档

%% kanban:settings
{"kanban-plugin":"board","lane-width":320}
%%
```

> ⚠️ 子看板三列：**待准备 → 直播后 → 归档完成**。无「直播中」列——直播时导演只看剧本/单场摘要/速查手册。

**tag 命名规范**：`#s1-1`、`#s1-2`（全小写，季-场用连字符）

## 模式C：Dataview 追踪表

**位置**：`L5_场次工作区/S{X}/S{X}-{N}/场次追踪表.md`
**用途**：展示本场文件同步状态（与 Kanban 文件分离存放）

### 步骤
1. 确认目标场次文件夹路径
2. 创建独立的 Dataview 查询文件

### 输出格式
```markdown
---
场次: S{X}-{N}
layer: L5_场次工作区
type: 场次追踪表
---

# S{X}-{N}《{标题}》· 追踪表

> 📊 Dataview 自动聚合本场文件状态。

---

## 🔄 文件同步状态

\`\`\`dataview
TABLE WITHOUT ID
  file.link AS "文件",
  已同步L2 AS "已同步 L2",
  已同步全局伏笔总表 AS "已同步伏笔表"
FROM "L5_场次工作区/S{X}/S{X}-{N}"
WHERE 已同步L2 != null OR 已同步全局伏笔总表 != null
\`\`\`

---

## 📁 本场所有文件

\`\`\`dataview
TABLE WITHOUT ID
  file.link AS "文件",
  file.mtime AS "最后修改"
FROM "L5_场次工作区/S{X}/S{X}-{N}"
SORT file.name ASC
\`\`\`
```

## 模式D：完整套件（B + C）

依次创建：
1. `场次子看板.md`（模式B）
2. `场次追踪表.md`（模式C）

完成后输出摘要：
```
✅ 场次 S{X}-{N} 看板套件已创建

📁 L5_场次工作区/S{X}/S{X}-{N}/
  ├── 场次子看板.md    ← Kanban 工作流（4列）
  └── 场次追踪表.md    ← Dataview 同步状态
```

## 模式E：更新主看板

**用途**：新场次建立后，把卡片加入全局进度看板

### 步骤
1. 读取 `L5_场次工作区/📋场次进度看板.md`
2. 在 `## 🔴 规划中` 列末尾追加新卡片
3. 格式：`- [ ] **S{X}-{N}**《{标题}》· {一句话描述} #{场次tag} #规划中`
4. 不修改其他任何内容

## Kanban ↔ Dataview 联动说明（Feature 3）

### 联动方式
- Kanban 卡片上的 `#tag` 可被 Dataview `TASK FROM` 查询捕获
- 全库仪表盘 `L0_导演作战台/04_全库仪表盘.md` 的「Kanban 同步视图」区块实时读取主看板任务

### 当前联动查询（已部署）
```dataview
TASK
FROM "L5_场次工作区/📋场次进度看板"
GROUP BY file.link
```

### 跨场次任务聚合（可用于未来）
```dataview
TASK
FROM "L5_场次工作区"
WHERE !completed AND contains(tags, "直播后")
GROUP BY file.link
```

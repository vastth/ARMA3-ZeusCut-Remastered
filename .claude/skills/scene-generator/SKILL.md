# 场次生成技能（Scene Generator）

## 触发场景

| 用户说 | 执行模式 |
|--------|---------|
| `"新建场次 S1-2"` / `"建场 S2-3"` | → **模式A：建立场次文件夹结构** |
| `"生成剧本 S1-2"` / `"写下一场"` / `"续写"` | → **模式B：生成场次剧本** |
| `"完整新场次 S1-2"` | → **模式C：A+B 一次完成** |
| `"填写摘要 S1-1"` / `"更新单场摘要"` | → **模式D：补全单场摘要** |
| `"填写状态变更单 S1-1"` | → **模式E：填写状态变更单** |

## 模式A：建立场次文件夹结构

**用途**：只建文件夹和空白模板文件，不写剧本内容。直播结束后填写用。

### 步骤

1. 确认目标场次（如 `S1-2`）
2. 读取所有5个模板文件
3. 读取 L2 四个状态文件，提取本场开场快照
4. 读取上一场 `单场摘要.md` 的「下一场承接入口」
5. 创建文件夹，写入5个文件：

**`单场摘要.md`**：按模板填入，预填「本场开场冻结快照」（从L2提取），其余区块留空

**`状态变更单.md`**：按模板复制，frontmatter 填入场次号，`已同步L2: false`

**`伏笔推进记录.md`**：按模板复制，frontmatter 填入场次号，从活跃伏笔预填「本场可能涉及的伏笔」

**`导演复盘卡.md`**：按模板复制（全空，直播后填写）

**`直播偏差记录.md`**：按模板复制（全空，直播后填写）

**`场次子看板.md`**：调用 `kanban-dataview` 技能的模式B，创建工作流看板
- ⚠️ 严格遵守：frontmatter 只保留 `kanban-plugin: board`，settings JSON 裸写不加反引号
- tag 格式：`#s{季}-{场}`（全小写，如 `#s1-3`）

**`场次追踪表.md`**：调用 `kanban-dataview` 技能的模式C，创建 Dataview 追踪文件
- 替换路径中的 `S{X}/S{X}-{N}` 为实际场次路径

7. 更新主看板：在 `L5_场次工作区/📋场次进度看板.md` 的 `## 🔴 规划中` 列末追加新卡片：
   `- [ ] **S{季}-{场}**《{标题}》· {一句话描述} #{场次tag} #规划中`

8. 输出创建摘要：
```
✅ 场次 S1-X 文件结构已创建

📁 L5_场次工作区/S1/S1-X/
  ├── 单场摘要.md         ← 开场快照已预填
  ├── 状态变更单.md        ← 等待直播后填写
  ├── 伏笔推进记录.md      ← 相关伏笔已标注
  ├── 导演复盘卡.md        ← 等待直播后填写
  ├── 直播偏差记录.md      ← 等待直播后填写
  ├── 场次子看板.md        ← 工作流看板（4列）✅ 新增
  └── 场次追踪表.md        ← Dataview 追踪表 ✅ 新增

📋 主进度看板已更新：S1-X 卡片已加入「规划中」列

剧本文件未生成，如需生成请说「生成剧本 S1-X」
```

## 模式B：生成场次剧本

**用途**：生成 `S{季}-{场}：《标题》.md` 剧本主文件，基于当前库状态续写。

### 步骤

**第一步：读取素材**（并行读取以下文件）

```
必读：
  - L4_内容生产层/活跃伏笔.md
  - L2_当前状态层/人物当前状态.md
  - L2_当前状态层/主线支线推进状态.md
  - L2_当前状态层/地点当前状态.md
  - L2_当前状态层/未决事项.md

如果上一场存在：
  - L5_场次工作区/S{X}/S{X}-{N-1}/单场摘要.md（承接点）
  - L5_场次工作区/S{X}/S{X}-{N-1}/状态变更单.md（上场变化）
```

**第二步：提取创作约束**

从上述文件中提取：

```
承接点：上一场「下一场直接承接入口」的描述（最高优先级）
活跃伏笔：🔴极紧急 → 本场必须推进，🟡中期 → 本场可推进
人物可用性：只使用「✅ 可用」的NPC，❌不可用的不能出场
未决事项：本场可以解决哪些悬而未决的问题
地点：本场可能使用的地点（控制权是否支持行动）
```

**第三步：规划场次结构**

生成前先输出规划摘要，等待用户确认（除非用户说"直接写"）：

```
## 场次规划草案：S{X}-{N}

建议标题：《》
核心主题（一句话）：
承接自：S{X}-{N-1} 的「[承接点描述]」

---

【本场推进的伏笔】
- 🔴 S1-F-001 [伏笔名]：计划推进方式
- 🟡 S1-F-007 [伏笔名]：可能触及

【本场出场NPC】
- 小蛋（主角）
- [NPC名]（功能/关系）

【场景规划】
- 第一幕：[幕名] — [地点] — [核心事件]
- 第二幕：[幕名] — [地点] — [核心事件]
- 第三幕：[幕名] — [地点] — [核心事件 + 悬念收尾]

【本场结尾悬念】：[下一场的钩子]

---
是否按此规划生成？或需要调整？
```

**第四步：生成剧本正文**

确认后，按以下格式生成 `S{季}-{场}：《标题》.md`：

````markdown
---
layer: L5_场次工作区
type: 单场剧本
场次: S{X}-{N}
标题: 《》
执行状态: 规划中
last_updated: {今日日期}
---

#### **第一幕：[幕名]**
- **开场**：[场景描述，氛围渲染，1-2句]
- **地点**：[地点名]
- [关键事件1]：[描述]
- [关键事件2]：[描述]
- **关键NPC**：[NPC名]（[本幕功能]）

---

#### **第二幕：[幕名]**

##### **[场景名]**
- **地点**：[地点]
- **事件**：[详细描述]
- **关键NPC**：[名字]（[功能]）
- **关键信息/情报**：[本场透露的信息]
- **选择分支**（如有）：
  - ✅ [选项A] → [结果A]
  - [选项B] → [结果B]
- **收获**：[行动结果]

---

##### **[场景名2]**
（同上格式）

---

#### **第三幕：[幕名]**

[同上格式]

---

**本场积累成果**
- ✅ [成果1]
- ✅ [成果2]
- [悬而未决的问题]

**关键设定**
- [世界观/人物关系的新信息]
````

**格式约束**：
- 与 S1-1 保持相同风格：幕→场景→事件的层级结构
- 选择分支用 `✅` 标注推荐选项
- 每个场景必须注明「关键NPC」和「地点」
- 结尾必须有「本场积累成果」和「关键设定」
- 视角信息用 `[已知]`/`[未知]`/`[推演]` 标注

## 模式C：完整新场次（A + B）

依次执行模式B（生成剧本）→ 模式A（建文件夹并预填摘要）。

建议流程：
1. 先展示规划草案（模式B第三步）
2. 用户确认后：生成剧本 + 建立所有辅助文件

## 模式D：填写单场摘要

**触发**：直播结束后，用户说「填写摘要 S1-1」

### 步骤

1. 读取 `L5_场次工作区/S{X}/S{X}-{N}/S{X}-{N}：《》.md`（原剧本）
2. 读取 `L5_场次工作区/S{X}/S{X}-{N}/直播偏差记录.md`（如已填写）
3. 读取 `L5_场次工作区/S{X}/S{X}-{N}/单场摘要.md`（当前状态）
4. 如有偏差记录，以**实际直播内容**为准（偏差 > 剧本规划）
5. 填写：
   - 「本场事件流」：按实际发生的事件压缩，不超过15条
   - 「关键转折点」：最重要的1-3个转折
   - 「下一场直接承接入口」：承接点（一句话）+ 悬念（2-3条）
6. 更新 `执行状态: 已完成`

---

## 模式E：填写状态变更单

**触发**：直播结束后，用户说「填写状态变更单 S1-1」

### 步骤

1. 读取该场剧本 + 直播偏差记录（如有）
2. 读取 L2 四个当前状态文件（作为对比基准）
3. 提取所有「变更」（前→后），按表格格式列出
4. 填写状态变更单四个区块（人物/地点/组织/资源）
5. 询问用户：**是否立即同步到L2？**
   - 若是：逐一修改 L2 文件，完成后把 `已同步L2: true`
   - 若否：保留为 false，后续用 `arma3-sync-audit` 统一同步

## 创作原则

### 叙事风格约束
- 参照 S1-1《换皮》的文风：**结构清晰、事件驱动、留有选择分支**
- 每场必须有**一个核心反转或道德困境**（参照「黑吃黑反转」）
- 伏笔推进必须**自然融入事件**，不能生硬插入
- NPC 对话和行为要符合**人物档案中的性格设定**

### 内容约束（禁止事项）
1. **不能让已死NPC出场**（检查L1/人物档案和L2/人物当前状态）
2. **不能违背世界规则**（读取 L1/世界规则.md）
3. **不能让 `[未知]` 视角的信息以 `[已知]` 方式透露**（如：小蛋视角不应该"知道"奥利弗的真实身份）
4. **不能凭记忆编造NPC名字**（以人物档案为准）
5. **伏笔推进必须符合ID规范**：引用时用 `SX-F-XXX`

### 伏笔使用规则
```
🔴 极紧急 → 本场必须有实质推进，不能只是"提及"
🟡 中期   → 可以埋下触发条件，不需要正面推进
🔵 长线   → 只能作为背景色，不主动推进
休眠     → 不出现，除非有特殊激活指令
```

```
格式：S{季}-{场}
  正常场次：S1-1, S1-2, S1-3 ...
  季末场次：S1末（特殊标记）
  支线场次：S1-外传-1（如有）

文件夹路径：L5_场次工作区/S{季}/S{季}-{场}/
剧本文件名：S{季}-{场}：《{标题}》.md
```

# ARMA3 知识库同步审核技能

## 触发场景

- 用户说"脚本更新了，同步库"
- 用户说"审核当前库是否对齐"
- 用户说"根据仲裁材料更新各模块"
- 用户提供新的剧情脚本内容

---

## 仲裁材料位置

```
L5_场次工作区/原始仲裁材料/完整版剧情脚本.md  ← 最高权威，以此为准
L5_场次工作区/原始仲裁材料/聊天记录剧情脚本_提取.md  ← 补充材料
```

| 层级 | 文件 | 职责 |

## 执行流程

### 第一步：读取仲裁材料

```
读取 完整版剧情脚本.md（全文）
提取关键变化：
  - 新增事件节点
  - 人物状态变化（生死/阵营/位置）
  - 新出现的NPC
  - 已兑现的伏笔
  - 新埋设的伏笔
  - 势力关系变化
  - 地点控制权变化
```

### 第二步：并行读取所有受管模块

一次性读取全部10个模块文件，建立当前库状态快照。

### 第三步：交叉比对，生成差异清单

对比仲裁材料与库状态，按以下维度检查：

**人物维度**
- [ ] 生死状态是否正确
- [ ] 阵营/立场是否最新
- [ ] 新NPC是否已录入
- [ ] 电话/代号等基础信息是否完整

**时间线维度**
- [ ] 事件节点是否完整、无冗余
- [ ] 季度标记是否正确（进行中/已完结）
- [ ] 未完成事项是否最新

**伏笔维度**
- [ ] 已兑现伏笔是否全部归档
- [ ] 活跃伏笔是否反映当前紧急度
- [ ] 新埋伏笔是否已录入

**势力/地点维度**
- [ ] 控制权变化是否同步
- [ ] 关系变化是否更新

### 第四步：执行修正

按差异清单逐项修正，每次Edit前必须确认已读取该文件。

修正原则：
- **L1层**（事实层）：只记录已确认事实，不记录推演
- **L2层**（状态层）：只记录当前有效状态，过期状态删除
- **L4层**（活跃伏笔）：AI续写专用，只保留本季必须兑现的伏笔
- **视角标注**：`[已知]` / `[推演]` / `[未知]` 必须准确

### 第五步：验证闭环

修正完成后，检查以下一致性：
- [ ] 同一人物在所有文件中状态一致
- [ ] 同一事件在时间线和伏笔档案中描述一致
- [ ] L0速查表的场次标记与L2同步
- [ ] 无"二当家"等已死人物被误写为活跃

---

## 常见错误模式（历史教训）

| 错误类型 | 示例 | 正确做法 |
|---------|------|---------|
| 已死人物仍标活跃 | "二当家：东部活动" | 改为"二当家残党：东部活动" |
| 已解决事项未清除 | "阿青被控制" | 改为"阿青已获救" |
| 伏笔分类错误 | 已兑现伏笔留在待兑现 | 移至已兑现列表 |
| 时间线冗余条目 | 同一事件出现两次 | 合并，保留更详细的 |
| 新NPC未录入 | 加百列、黑凯撒等 | 补充到配角表 |
| 阵营描述过时 | "阿南：盟友" | 改为"阿南：叛变，敌对" |

## 输出格式

审核完成后输出：

```
## 本轮同步摘要

**新增场次**：S4-X（如有）
**关键状态变化**：
- 人物：XXX → XXX
- 伏笔：XXX 已兑现 / XXX 新埋设

**修正文件**：X个模块
**当前剧情位置**：第X季，小蛋位于XXX
```

---

## 注意事项

1. 仲裁材料为最高权威，库中任何与脚本矛盾的内容以脚本为准
2. 导演速查手册（L3层）不在自动同步范围，需用户单独指令
3. 场次编号格式：`S{季}-{场}` 或 `S{季}末`
4. 视角标注不可省略，`[推演]` 表示未在脚本中明确确认的内容

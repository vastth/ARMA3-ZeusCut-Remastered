# 剧情一致性检查技能（Consistency Checker）

## 触发场景

| 用户说 | 执行模式 |
|--------|---------|
| "查矛盾" / "一致性检查" / "库是否自洽" | → **全量检查模式** |
| "查人物矛盾" | → **人物专项检查** |
| "查伏笔逻辑" | → **伏笔专项检查** |
| "查时间线" | → **时间线专项检查** |
| "查势力关系" | → **势力/地点专项检查** |

---

## 文件地图（检查范围）

### L1 事实层（权威源）
```
L1_事实层/人物档案.md           ← 人物基础信息、生死状态、阵营
L1_事实层/时间线.md             ← 已发生事件节点
L1_事实层/伏笔档案.md           ← 伏笔分类（待兑现/已兑现/冲突）
L1_事实层/势力档案.md           ← 势力结构
L1_事实层/地点档案.md           ← 地点归属
```

### L2 状态层（当前快照）
```
L2_当前状态层/人物当前状态.md    ← 实时人物状态
L2_当前状态层/主线支线推进状态.md ← 剧情进度
L2_当前状态层/地点当前状态.md    ← 地点控制权
L2_当前状态层/组织关系当前状态.md ← 势力关系
L2_当前状态层/未决事项.md        ← 待处理问题
```

### L4 内容生产层（伏笔体系）
```
L4_内容生产层/活跃伏笔.md
L4_内容生产层/休眠伏笔.md

```

## 六类矛盾检测规则

### 检测A：死亡人物一致性

**读取**：`L1_事实层/人物档案.md` 中标注「已牺牲/已死亡/已消灭」的人物

**检查范围**：
- `L2_当前状态层/人物当前状态.md` 中该人物是否仍标「活跃」
- `L4_内容生产层/活跃伏笔.md` 中是否有该人物作为「关键NPC」且状态为进行中
- `L2_当前状态层/组织关系当前状态.md` 中是否仍将其列为现任职位

**已知问题模式（历史教训）**：
```
错误：约翰·谢尔比（二当家）死亡后，伏笔「S1-F-005 二当家野心」仍标「活跃-长线」
正确：移入「已回收伏笔归档」或注明「该伏笔随NPC死亡终止」

错误：史哥牺牲后，人物当前状态仍写「史哥：活跃，剃刀党三号人物」
正确：改为「史哥：已牺牲（SX-X场）」
```

---

### 检测B：人物状态跨文件一致性

**核心矛盾对**：

| 检查对象 | 文件A | 文件B | 检查内容 |
|---------|-------|-------|---------|
| 人物阵营 | L1/人物档案 | L2/人物当前状态 | 阵营描述是否一致 |
| 人物位置 | L2/人物当前状态 | L2/地点当前状态 | 人物所在地点是否与地点控制权吻合 |
| 人物可用性 | L2/人物当前状态「可用性」 | L2/未决事项 | 被标「❌ 不可用」的人物是否出现在待处理事项中 |

**重点检查**：
- 阵营叛变后（如「阿南：盟友 → 叛变，敌对」），所有文件是否同步更新
- 视角标注是否一致：同一人物在不同文件中 `[已知]/[未知]` 是否矛盾

---

### 检测C：时间线逻辑

**读取**：`L1_事实层/时间线.md`

**检查规则**：

| 问题类型 | 判断依据 |
|---------|---------|
| 事件顺序矛盾 | A事件发生在B事件之后，但B事件依赖A事件的结果 |
| 冗余条目 | 同一事件出现两次以上（措辞不同但内容相同） |
| 状态标注错误 | 「🔴 进行中」的季度，但主线推进状态文件标该季「已完结」 |
| 孤立条目 | 时间线中有事件节点，但L2层无任何对应状态变化 |

**输出格式**：
```
❌ 时间线矛盾 [严重度]
  位置：第X季 第X条
  问题：[描述]
  建议：[修正方向]
```

---

### 检测D：伏笔体系完整性

**读取**：L1/伏笔档案 + L4三个文件

**检查规则**：

| 问题类型 | 判断依据 |
|---------|---------|
| L1与L4脱节 | L1「待兑现」中有伏笔，但在L4活跃/休眠中均找不到对应ID |
| 已兑现伏笔残留 | 伏笔在L1「已兑现」中，但L4「活跃伏笔」中仍存在 |
| 视角标注缺失 | 任一伏笔条目缺少 `[已知]/[未知]/[推演]` |
| ID格式不规范 | 不符合 `SX-F-XXX` 格式 |
| 关键NPC已死但伏笔未处理 | 伏笔依赖已死人物，但伏笔状态仍为「进行中」 |
| 跨层重复 | 同一伏笔同时出现在「活跃」和「休眠」 |

---

### 检测E：势力与地点一致性

**读取**：L1/势力档案 + L1/地点档案 + L2/组织关系当前状态 + L2/地点当前状态

**检查规则**：

| 问题类型 | 判断依据 |
|---------|---------|
| 地点控制权矛盾 | L1/地点档案 与 L2/地点当前状态描述不一致 |
| 已灭亡势力仍活跃 | L1/势力档案 标「已消灭」，但L2/组织关系中仍有该势力条目 |
| 关系变化未同步 | L2/组织关系当前状态 中关系标注与最新场次剧情不符 |

---

### 检测F：主线支线推进一致性

**读取**：L2/主线支线推进状态 + L1/时间线 + L0速查表（若存在）

**检查规则**：

| 问题类型 | 判断依据 |
|---------|---------|
| 场次标记不同步 | L2中「当前场次」与时间线中「进行中」标记不一致 |
| 支线悬空 | 支线标「进行中」，但无任何对应人物/地点状态支撑 |
| 已完结主线未归档 | 主线标「进行中」但时间线中对应季已标「已完结」 |

## 修正执行原则

发现矛盾后，按以下优先级决定「以哪个文件为准」：

```
优先级（高→低）：
1. L5_场次工作区/原始仲裁材料/完整版剧情脚本.md（最高权威）
2. L1_事实层/（事实层，稳定基准）
3. L2_当前状态层/（状态层，实时快照）
4. L4_内容生产层/（内容层，从属于L1）
```

**修正时**：
- 以高优先级文件内容为准
- 修正低优先级文件，而非修改高优先级文件
- 每次Edit前必须重新读取目标文件
- 修正后在报告中标注「已修正」

---

## 快速专项检查（轻量版）

用户只想检查某一类时，只读取相关文件：

| 专项 | 读取文件 | 执行检测 |
|------|---------|---------|
| 人物专项 | L1/人物档案 + L2/人物当前状态 | 检测A + 检测B |
| 伏笔专项 | L1/伏笔档案 + L4三文件 | 检测D |
| 时间线专项 | L1/时间线 + L2/主线支线推进状态 | 检测C + 检测F |
| 势力地点专项 | L1/势力+地点档案 + L2/组织+地点状态 | 检测E |

---

## 常见误报排除

以下情况**不是矛盾**，不要误报：

| 情况 | 原因 |
|------|------|
| L2中人物标 `[未知]` 但L1有记录 | 视角不同，L1是导演视角，L2是剧情内视角 |
| 时间线中有「剧情脚本参考」条目 | 这是预设走向，非已发生事实 |
| 休眠伏笔中的NPC当前不活跃 | 休眠伏笔等待激活条件，不要求关联人物活跃 |
| L1/人物档案中「状态：活跃（重制版起点，尚未牺牲）」 | 这是重制版预告信息，不与L2矛盾 |

## 检查完成后

输出最终评级：

```
## 库健康度评级

🟢 健康（0个严重矛盾）：可以放心推进下一场
🟡 轻微问题（仅有轻微不一致）：建议修正后推进
🟠 中等风险（有中等问题）：建议优先修正再续写
🔴 高风险（有严重矛盾）：必须修正，否则续写会产生错误

当前评级：[X]
```

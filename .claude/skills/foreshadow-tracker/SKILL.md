# 伏笔追踪技能（Foreshadow Tracker）

## 触发场景

| 用户说 | 执行模式 |
|--------|---------|
| "查伏笔" / "伏笔审查" / "伏笔现状" | → **全量扫描模式** |
| "兑现伏笔 SX-F-XXX" | → **兑现操作模式** |
| "激活伏笔 SX-F-XXX" | → **激活操作模式** |
| "新增伏笔" / "埋设伏笔" | → **录入模式** |
| "查孤立伏笔" / "有没有被遗忘的伏笔" | → **孤立检测模式** |
| "伏笔统计" / "伏笔总览" | → **统计报告模式** |

---

## 文件地图

```
L1_事实层/伏笔档案.md          ← 事实层，主分类存档（待兑现/已兑现/冲突）
L4_内容生产层/全局伏笔总表.md  ← 导航索引，ID规范说明
L4_内容生产层/活跃伏笔.md      ← 当前正在推进，本季必须兑现
L4_内容生产层/休眠伏笔.md      ← 已埋设但暂不激活
L4_内容生产层/已回收伏笔归档.md ← 已兑现，永久存档
```

**操作优先级**：ID规范以 `全局伏笔总表.md` 为准，格式为 `SX-F-XXX`。

## 模式一：全量扫描（最常用）

### 步骤

1. 并行读取所有五个伏笔文件
2. 按以下维度输出报告：

```
## 伏笔全景报告

### 🔴 极紧急 - 本季必须兑现
（来自 活跃伏笔.md 🔴 区块）
列出：ID / 伏笔内容 / 当前状态 / 预计回收 / 最后更新

### 🟡 中期激活 - 近期可能触发
（来自 活跃伏笔.md 🟡 区块）

### 🔵 长线伏笔 - 暂不急
（来自 活跃伏笔.md 🔵 区块）

### 😴 休眠中
（来自 休眠伏笔.md，列出激活条件）

### ✅ 已回收（本季）
（来自 已回收伏笔归档.md，只显示最近一季）

### ⚠️ 冲突伏笔
（来自 L1_事实层/伏笔档案.md 冲突区块）

---
活跃总数：X  |  休眠总数：X  |  已回收总数：X
```

## 模式二：兑现操作

**触发**：用户说 "兑现伏笔 S1-F-001" 或 "S1-F-001 已兑现"

### 步骤

1. 读取 `活跃伏笔.md`，定位该ID条目
2. 读取 `已回收伏笔归档.md`，了解现有格式
3. 读取 `L1_事实层/伏笔档案.md`，定位待兑现区块
4. 询问用户：**兑现场次是哪场？** （如 S1-3）
5. 执行三步修改：

**Step A** — 从 `活跃伏笔.md` 中删除该条目行

**Step B** — 在 `已回收伏笔归档.md` 中追加归档条目：
```
| SX-F-XXX | 伏笔内容 | 埋设场次 | 兑现场次 | 兑现方式（一句话） |
```

**Step C** — 在 `L1_事实层/伏笔档案.md` 的「已兑现伏笔」区块追加记录

6. 输出确认摘要：
```
✅ 伏笔 SX-F-XXX 已兑现
- 从：活跃伏笔 🔴 极紧急
- 至：已回收伏笔归档（兑现场次：SX-X）
- 修改文件：3个
```

---

## 模式三：激活操作

**触发**：用户说 "激活伏笔 M1" 或 "把威廉的画移到活跃"

### 步骤

1. 读取 `休眠伏笔.md`，定位该条目
2. 读取 `活跃伏笔.md`，了解现有格式和最大ID
3. 询问用户：**激活为哪个优先级？** 🔴 / 🟡 / 🔵
4. 执行两步修改：

**Step A** — 从 `休眠伏笔.md` 删除该条目

**Step B** — 在 `活跃伏笔.md` 对应优先级表格中追加，补充字段：
- `最后更新`：当前场次标记
- 保留原有 `视角` 标注

5. 输出确认摘要

---

## 模式四：录入新伏笔

**触发**：用户说 "新增伏笔" / "埋设伏笔"

### 步骤

1. 读取 `全局伏笔总表.md`，确认ID规范
2. 读取 `活跃伏笔.md`，找出当前最大 ID 序号
3. 生成新 ID：`SX-F-XXX`（X为下一个序号）
4. 向用户确认以下信息（若未提供则询问）：

```
- 伏笔内容：
- 埋设场次：
- 当前状态：
- 预计回收：
- 视角标注：[已知] / [未知] / [推演]
- 分层：活跃（哪个优先级）/ 休眠（激活条件是什么）
```

5. 写入对应文件
6. 若为 🔴 极紧急，同步在 `L1_事实层/伏笔档案.md` 「待兑现」区块中添加

---

## 模式五：孤立检测

**触发**：用户说 "查孤立伏笔" / "有没有被遗忘的伏笔"

### 步骤

1. 并行读取所有五个伏笔文件
2. 检测以下异常：

| 异常类型 | 判断依据 |
|---------|---------|
| 超期未兑现 | `最后更新` 字段停留在2季以前的场次，且状态未变化 |
| L1与L4不一致 | 伏笔在L1「待兑现」存在，但L4三个文件中均找不到对应条目 |
| 视角标注缺失 | 条目中无 `[已知]` / `[未知]` / `[推演]` 任一标注 |
| ID格式错误 | 不符合 `SX-F-XXX` 格式的条目 |
| 已死人物关联 | 伏笔关键NPC为已确认死亡人物，但伏笔仍标「活跃」 |

3. 输出问题清单，按严重程度排序
4. 询问用户是否逐项处理

**触发**：用户说 "伏笔统计" / "伏笔总览"

输出格式：
```
## 伏笔健康度报告

| 分层 | 数量 | 详情 |
|------|------|------|
| 🔴 活跃-极紧急 | X | - |
| 🟡 活跃-中期 | X | - |
| 🔵 活跃-长线 | X | - |
| 😴 休眠 | X | - |
| ✅ 已回收 | X | - |
| ⚠️ 冲突 | X | - |

**健康度评估**：
- 极紧急伏笔过多（>5）→ ⚠️ 负担过重，建议降级或合并
- 休眠伏笔超期（>3季未激活）→ ⚠️ 检查是否需要清理
- 已回收比例 → 剧情推进进度参考
```

---

## 操作禁忌

1. **绝不凭记忆操作**：每次操作前必须先读取相关文件
2. **不得改变ID**：兑现/激活时保留原始 `SX-F-XXX`，只移动不重编
3. **视角标注必须保留**：移动条目时 `[已知]/[未知]/[推演]` 不可丢失
4. **L1和L4必须同步**：对 🔴 极紧急伏笔的操作必须同步更新L1层

---

## 快速参考：字段说明

| 字段 | 说明 |
|------|------|
| ID | `SX-F-XXX`，S=季，F=伏笔，XXX=序号 |
| 埋设场次 | 首次出现的场次，如 `S1-1场景3` |
| 当前状态 | 一句话描述当前进展 |
| 预计回收 | 预计兑现的时间范围 |
| 最后更新 | 最近一次修改的场次标记 |
| 视角 | `[已知]`=导演视角已知 / `[未知]`=导演视角未知 / `[推演]`=未在脚本确认 |
